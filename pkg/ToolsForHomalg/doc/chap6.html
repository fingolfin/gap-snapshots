6 ToDo-list

6.1 ToDo-list entries

6.1-1 SourcePart
‣ SourcePart( entry )( operation )

Returns: a list

Returns the a list of source parts of the ToDo-list entry entry. This is a triple of an object, a name of a filter/attribute\ , and a value to which the attribute has to be set to activate the entry

6.1-2 TargetPart
‣ TargetPart( entry )( operation )

Returns: a list

Returns the target part of the ToDo-list entry entry. This is a triple of an object, a name of a filter/attribute, and a val\ ue to which the specific filter/attribute should be set.

6.1-3 ProcessAToDoListEntry
‣ ProcessAToDoListEntry( arg1 )( operation )

Returns: a boolean

Processes a ToDo-list entry, i.e. sets the information given in TargetPart if the definitions in SourcePart are fulfilled. Returns \ true if the entry could be processed, false if not, and fail if SourcePart or TargetPart weren't availible anymore.

6.1-4 AreCompatible
‣ AreCompatible( arg1, arg2 )( operation )

Returns: a boolean

Returns true if TargetPart( A ) = SourcePart( B ), false otherwise.

6.1-5 ToDoListEntryWithWeakPointers
‣ ToDoListEntryWithWeakPointers( arg1, arg2, arg3, arg4, arg5, arg6 )( operation )

Returns: a ToDoListEntry

Creates a ToDo-list entry where the first three arguments correspond to the SourcePart of the entry and the last three ones to the \ TargetPart. This type of entry uses weakpointers as links to the object so you might be careful with it.

6.1-6 ToDoListEntryWithPointers
‣ ToDoListEntryWithPointers( arg1, arg2, arg3, arg4, arg5, arg6 )( operation )

Returns: a ToDoListEntry

The same as ToDoListEntryWithWeakPointers, but with pointers.

6.1-7 ToDoListEntryWithListOfSources
‣ ToDoListEntryWithListOfSources( arg1, arg2, arg3, arg4 )( operation )

Returns: a ToDoListEntry

The same as ToDoListEntryWithPointers, but the first argument must be a list of triples defining the conditions of this entry

6.1-8 ToDoListEntryWhichLaunchesAFunction
‣ ToDoListEntryWhichLaunchesAFunction( arg1, arg2 )( operation )

Returns: a ToDoListEntry

The first argument must be a list of triples which contain an object, on which the second entry, an attribute given by its name as \ a string can be applied. If the value of the attribute matches the third entry, the function given as second argument is launched.

6.1-9 CreateImmediateMethodForToDoListEntry
‣ CreateImmediateMethodForToDoListEntry( arg1 )( operation )

Returns: nothing

Creates an ImmediateMethod for a ToDoListEntry. This method is for internal use only.

6.1-10 JoinToDoListEntries
‣ JoinToDoListEntries( arg1 )( operation )

Returns: a ToDoListEntry

Joins a list of compatible ToDo-list entries to a new one.

6.1-11 GenesisOfToDoListEntry
‣ GenesisOfToDoListEntry( arg )( attribute )

Returns: a list

If a ToDoListEntry is created as union of others, this is a list of those.

6.1-12 DescriptionOfImplication
‣ DescriptionOfImplication( arg )( attribute )

Returns: a list

Has to be set to a string, which describes the reason for the conclusion. If the ToDo-list entry is displayed, the given string wil\ l be displayed with a because before it.

6.2 Category

6.2-1 IsToDoList
‣ IsToDoList( arg )( category )

Returns: true or false

This is the category of ToDo-lists. Every ToDo-list is an object of this category, which basically contains the ToDo-lists.

6.3 Constructor

6.3-1 NewToDoList
‣ NewToDoList( )( operation )

Returns: nothing

Creates a new empty ToDo-list.

6.4 This is the magic

6.4-1 Process_A_ToDo_List_Entry
‣ Process_A_ToDo_List_Entry( args )( function )

Returns: a boolean

Gets a ToDo-list entry, which is a pair of a list of strings and a weak pointer object, and processes it. If the action was done, i\ t returns true, if not, it returns false, and it returns fail if the action is not possible anymore due to deleted objects.

6.4-2 ProcessToDoList
‣ ProcessToDoList( A )( attribute )

Returns: nothing

This is the magic! This attribute is never set. Creating an ToDo-list entry installs an ImmediateMethod for this attribute for the \ specific category of the object to which ToDo-list is added, and the filter the entry contains. It is then triggert if the filters become a\ pplicable, so the ToDo-list is processed

6.5 Methods for all objects

6.5-1 AddToToDoList
‣ AddToToDoList( E )( operation )

Returns: Nothing

Adds the ToDo-list entry E to the ToDo-list of M and creates a new one, if this is needed.

6.5-2 ToDoList
‣ ToDoList( arg )( attribute )

Returns: A ToDo-list

Returns the ToDo-list of an object, or creates a new one.
